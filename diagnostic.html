🔑 Flujo de Autenticación

El sistema utiliza autenticación en dos capas:

Capa 1: Cliente → Express (Local)

El cliente envía credenciales al servidor Express que actúa como proxy.

POST /api/get HTTP/1.1 Host: localhost:5000 Content-Type: application/json { "action": "TryLoginFacilities", "email": "facility1@wounddatacenter.com", "password": "***", "deviceId": "web-abc123" }

Capa 2: Express → Backend Remoto

Express reenvía las credenciales al backend remoto que realiza la validación.

POST https://cubed-mr.app/api/get HTTP/1.1 Content-Type: application/json { "entity": "TryLoginFacilities", "email": "facility1@wounddatacenter.com", "password": "***", "deviceId": "web-abc123" }

✗ Respuesta fallida (status === 0)
