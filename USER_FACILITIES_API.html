🏥 User Facilities API

Endpoint para listar facilities accesibles por usuario

📋 Descripción

Endpoint que retorna la lista de facilities a los que un usuario tiene acceso.

Al hacer login, el sistema cachea automáticamente el array de facilities asociadas al email del usuario. Este endpoint permite consultar esa información posteriormente.

✨ Características

🚀 Cache Automático

Se actualiza automáticamente en cada login

🔍 Flexible

GET con query params o POST con body

⚡ Instantáneo

Consulta rápida desde cache en memoria

📦 Completo

Retorna facilities, entityId y entityName

🔌 Endpoints

GET /api/user/facilities

Query Parameters:

?email=facility@example.com

Response:

{ "status": true, "email": "facility1@wounddatacenter.com", "facilities": [1, 2, 3, 5], "entityId": "123", "entityName": "Main Facility Group", "cached": true }
POST /api/user/facilities

Request Body:

{ "email": "facility@example.com" }

Response:

{ "status": true, "email": "facility@example.com", "facilities": [1, 2, 3], "entityId": "456", "entityName": "Healthcare Network", "cached": true }

📊 Flujo de Datos

User LOGIN (/api/get) ↓ System Cache UPDATE email → {facilities, entityId, entityName} ↓ GET /api/user/facilities?email=... ↓ Return facilities array from cache

📦 Campos de Respuesta

Campo Tipo Descripción
status boolean Indica si la solicitud fue exitosa
email string Email del usuario consultado
facilities array Array de IDs de facilities accesibles
entityId string/number ID de la entidad (facility group)
entityName string Nombre de la entidad
cached boolean Indica si los datos vinieron del cache

💻 Uso en Frontend (React)

Hook personalizado:

import { useUserFacilities, useHasFacilityAccess } from "@/hooks/use-user-facilities"; function MyComponent() { // Obtener lista de facilities const { data, isLoading, error } = useUserFacilities(); if (data?.facilities) { console.log("Facilities:", data.facilities); } // Verificar acceso a facility específica const hasAccess = useHasFacilityAccess("5"); if (hasAccess) { console.log("User can access facility 5"); } }

Componentes listos para usar:

import { UserFacilitiesList, FacilityAccessOverview, FacilityAccessMatrix } from "@/components/user-facilities"; export function Dashboard() { return ( <> <FacilityAccessOverview /> <UserFacilitiesList /> <FacilityAccessMatrix facilities={[1, 2, 3, 5]} /> </> ); }

🧪 Pruebas

Con curl (GET):

curl -X GET "http://localhost:5000/api/user/facilities?email=facility1@wounddatacenter.com" \ -H "Content-Type: application/json"

Con curl (POST):

curl -X POST "http://localhost:5000/api/user/facilities" \ -H "Content-Type: application/json" \ -d '{"email":"facility1@wounddatacenter.com"}'

Script completo (Node.js):

node /var/www/facility/test/test-user-facilities.js

🔧 Detalles de Implementación

Cache en memoria:

const userFacilitiesCache = new Map<string, { facilities: string[] | number[]; entityId: string | number; entityName: string; timestamp: number; }>();

Se actualiza en login:

if (data.status === 1) { userFacilitiesCache.set(email, { facilities: data.data?.[0]?.facilities || [], entityId: data.data?.[0]?.entityId || "", entityName: data.data?.[0]?.entityName || "", timestamp: Date.now(), }); }

⚠️ Nota: El cache se pierde si el servidor reinicia. Para producción, considerar migrar a Redis o base de datos.
