🔑 FLUJO DE AUTENTICACIÓN

  1. Usuario envía email y contraseña en formulario login
  
  2. JavaScript envía:
     {
       action: "TryLoginFacilities",
       email: "facility@example.com",
       password: "***",
       deviceId: "web-xxxxx"
     }
  
  3. Express valida y reenvía al backend remoto (HTTPS)
  
  4. Backend valida contra SQL Server y retorna:
     {
       status: 1,                    // 1 = éxito, 0 = fallo
       token: "JWT...",              // Token de sesión
       entityId: 123,                // ID de facility
       entityName: "Facility Name",
       facilities: [1, 2, 3],        // Facilities accesibles
       msg: "Success"
     }
  
  5. Cliente almacena en localStorage:
     - authToken (JWT)
     - userFacilityId
     - userEmail
     - userEntityName
     - userFacilities
